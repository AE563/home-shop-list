# Tech Stack: Django + DRF + Vanilla JS + Bootstrap 5

## Backend — ядро

**Django 5.1** — основной фреймворк. ORM, миграции, админка, шаблоны, middleware, система аутентификации — всё из коробки. Зрелый, стабильный, огромное комьюнити.

**Django REST Framework (DRF) 3.15** — надстройка над Django для создания REST API. Сериализаторы, вьюсеты, роутеры, пагинация, фильтрация, throttling, аутентификация по токенам. Без него писать API на Django — боль.

**django-filter 24.3** — фильтрация queryset'ов в DRF. Вместо ручного разбора query-параметров описываешь `FilterSet` — и получаешь фильтрацию, поиск, сортировку из коробки. Стандартный компаньон DRF.

**drf-spectacular 0.27** — автогенерация OpenAPI/Swagger документации из DRF. Описания эндпоинтов, параметров, ответов — всё строится автоматически по сериализаторам и вьюсетам. Swagger UI доступен прямо в браузере.

**requests 2.32** — библиотека для исходящих HTTP-запросов. Нужна для отправки вебхуков: событие произошло → твой сервер сам шлёт POST на внешний URL. Простой и удобный API.

## База данных

**SQLite** — для локальной разработки. Не нужно ничего устанавливать и настраивать, файл базы создаётся автоматически. Идеально для быстрого старта и прогона тестов.

**PostgreSQL 16** — для прода. Надёжная, быстрая, поддерживает полнотекстовый поиск, JSONB, массивы, транзакции. Django ORM работает с ней без проблем. Переключение между SQLite и PostgreSQL — одна строка в `DATABASES` настройках.

**psycopg 3.2** — драйвер для подключения Django к PostgreSQL. Третья версия — асинхронная, быстрее и современнее старого psycopg2. Ставится только для прода/стейджинга.

## Frontend

Фронтенд и бэкенд живут в одном проекте — Django отдаёт шаблоны, статика лежит в `static/`.

**Vanilla JavaScript (ES6+)** — без фреймворков. Fetch API для запросов к бэкенду, работа с DOM напрямую. Для проектов малого и среднего размера — проще, быстрее, нет лишних зависимостей. Легко понять, что происходит в коде.

**Bootstrap 5.3** — CSS-фреймворк для UI. Готовая сетка, компоненты (модалки, карточки, навбар, формы, алерты), адаптивность из коробки. Пятая версия убрала jQuery — отлично сочетается с ванильным JS. Подключается через CDN, ничего не нужно собирать.

## Тестирование

**pytest 8.3** — основной тест-раннер. Проще и мощнее стандартного unittest: фикстуры, параметризация, плагины, понятный вывод ошибок. Пишешь обычные функции с assert — и всё работает.

**pytest-django 4.9** — интеграция pytest с Django. Даёт фикстуры вроде `client`, `db`, `admin_client`, настраивает `DJANGO_SETTINGS_MODULE`, управляет тестовой базой данных. Требует настройки в `pyproject.toml` (см. раздел «Конфигурация»).

**pytest-cov 5.0** — покрытие кода тестами. Показывает, какие строки не покрыты. Генерирует отчёты в терминал, HTML или XML. Помогает находить слепые зоны.

**factory-boy 3.3** — фабрики для создания тестовых объектов. Вместо ручного `User.objects.create(username="test", email="test@test.com", ...)` пишешь `UserFactory()` — и получаешь объект с реалистичными данными. Связанные объекты создаются автоматически.

**Faker 28.0** — генерация фейковых данных: имена, email, адреса, тексты, даты, телефоны. Работает в связке с factory-boy. Каждый запуск тестов — уникальные данные, что помогает ловить баги.

## Инструменты разработки

**django-debug-toolbar 4.4** — панель отладки прямо в браузере. Показывает SQL-запросы (сколько их и за какое время), использование кэша, шаблоны, сигналы, заголовки. Незаменим для поиска N+1 запросов и оптимизации.

**python-dotenv 1.0** — загрузка переменных окружения из `.env` файла. `SECRET_KEY`, `DATABASE_URL`, ключи API — всё хранится в `.env`, не попадает в git, легко менять между окружениями (dev/staging/prod).

## Безопасность и проверка уязвимостей

### Python / Django

**pip-audit** — сканирует установленные пакеты по базе уязвимостей (OSV, PyPI). Запустил `pip-audit` — получил список пакетов с известными CVE и рекомендации по обновлению. Легко встраивается в CI.

**Bandit** — статический анализатор безопасности Python-кода. Ищет типичные дыры: использование `eval()`, хардкод паролей, небезопасные SQL-запросы, слабые хэши, опасный `pickle`. Не просто проверяет зависимости, а сканирует твой собственный код.

**django-check --deploy** — встроенная команда Django. Проверяет настройки перед деплоем: включён ли HTTPS, стоит ли `SECURE_HSTS_SECONDS`, правильно ли настроены cookies, не забыт ли `DEBUG=False`. Бесплатно и из коробки.

### JavaScript

**npm audit** — встроенный в npm сканер. Проверяет дерево зависимостей на известные уязвимости. `npm audit fix` пытается автоматически обновить пакеты до безопасных версий.

**Snyk** — мощный сканер уязвимостей для всего стека. Проверяет и Python, и JS зависимости, интегрируется с GitHub/GitLab, шлёт уведомления о новых уязвимостях. Есть бесплатный план.

## Качество кода и линтинг

### Python

**Ruff** — сверхбыстрый линтер и форматтер для Python (на Rust). Заменяет сразу flake8, isort, black, pycodestyle и десяток плагинов. Один инструмент вместо пяти. Работает в 10–100 раз быстрее аналогов.

### JavaScript

**ESLint** — главный линтер для JavaScript. Находит ошибки, проблемы стиля, потенциальные баги. Для ванильного JS используй конфиг `eslint:recommended`. Правила настраиваются: хочешь строгие — включай `no-var`, `eqeqeq`, `no-unused-vars`. Можно добавить плагин `eslint-plugin-security` для проверки на XSS, инъекции и тому подобное.

**Prettier** — автоформаттер кода. Не ищет баги, а приводит код к единому стилю: кавычки, отступы, длина строк. Работает в паре с ESLint — один форматирует, другой проверяет логику.

## Логирование

**structlog** — структурированное логирование. Вместо строк `"User 123 logged in"` получаешь JSON с полями `{"event": "login", "user_id": 123, "ip": "..."}`. Легко парсить, фильтровать, отправлять в ELK/Sentry. Для продакшена это must have — без нормальных логов дебаг превращается в гадание.

## Конфигурация

### pyproject.toml (минимальная)

```toml
[tool.pytest.ini_options]
DJANGO_SETTINGS_MODULE = "config.settings"
python_files = ["tests.py", "test_*.py"]
python_paths = ["."]
addopts = "--cov=apps --cov-report=term-missing -v"

[tool.ruff]
line-length = 120
target-version = "py312"

[tool.ruff.lint]
select = ["E", "F", "I", "S", "B", "UP"]
# E/F — базовые ошибки, I — импорты, S — bandit-правила, B — баги, UP — modernize
```

## Быстрый старт

```bash
# 1. Виртуальное окружение
python -m venv .venv
source .venv/bin/activate

# 2. Зависимости
pip install -r requirements.txt

# 3. Переменные окружения
cp .env.example .env

# 4. База и миграции
python manage.py migrate

# 5. Запуск
python manage.py runserver

# 6. Тесты
pytest

# 7. Проверка безопасности
pip-audit                           # уязвимости в Python-пакетах
bandit -r apps/                     # уязвимости в твоём Python-коде
python manage.py check --deploy     # настройки Django перед деплоем
npx eslint static/js/               # линтинг JS
```
