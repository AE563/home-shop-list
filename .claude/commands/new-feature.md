# /new-feature — TDD-цикл для новой фичи

**Правило:** тесты пишутся ДО реализации. Не наоборот.

---

## Фаза 1 — Спланировать (Plan Mode)

Войти в plan mode. Определить:
- Что делает фича (ожидаемое поведение)?
- Какие файлы затронуты (модели, views, JS, тесты)?
- Нужен ли WebSocket broadcast?
- Есть ли граничные случаи (ошибки, пустые данные, права доступа)?

Добавить запись в `ARCHITECTURE.md → Feature Backlog` (дата, название, решение, файлы).

Получить ОК пользователя перед реализацией.

---

## Фаза 2 — Написать тесты (RED)

Написать тесты для **всех** запланированных случаев:
- Happy path (основной сценарий)
- Ошибки (400, 404, 403)
- Граничные случаи
- Auth (требует `@login_required`?)
- WebSocket broadcast (если нужен)

**Python:** `tests/test_api.py`, `tests/test_models.py`
**JS:** `tests/js/ui.test.js`, `tests/js/websocket.test.js`

Использовать существующие фикстуры из `tests/conftest.py`.

**Не реализовывать функциональность.** Тесты будут ссылаться на несуществующий код — это нормально.

---

## Фаза 3 — Убедиться что тесты падают

```bash
# Python
.venv2/bin/pytest --no-cov -q -k "имя_фичи"

# JavaScript
npm test -- --testNamePattern="имя фичи"
```

Тесты ОБЯЗАНЫ упасть (`ImportError`, `AttributeError`, `AssertionError`).
Если тест зелёный без реализации — он не проверяет нужное. Вернись на Фазу 2.

---

## Фаза 4 — Реализовать фичу

Реализуй в правильном порядке:

1. **Модель** (`apps/shop/models.py`) — новые методы или поля
2. **View** (`apps/shop/views.py`) — endpoint по шаблону из `django-senior`
3. **URL** (`config/urls.py`)
4. **Frontend JS** (`static/js/ui.js`) — event delegation, AJAX
5. **WebSocket** (`static/js/websocket.js`) — новый тип события (если нужен)

Следовать соглашениям из `django-senior` и `vanilla-js` skills.

---

## Фаза 5 — Убедиться что тесты проходят

```bash
.venv2/bin/pytest --no-cov -q -k "имя_фичи"
npm test -- --testNamePattern="имя фичи"
```

Все написанные тесты ОБЯЗАНЫ стать зелёными.

---

## Фаза 6 — Прогнать все тесты (регрессия)

```bash
.venv2/bin/pytest --no-cov -q
npm test
```

Если что-то сломалось — исправь до пуша.

---

## Фаза 7 — Обновить ARCHITECTURE.md

- Переставить задачу из `Осталось сделать` в `Готово` (или в `Feature Backlog` как done)
- Обновить статус в Feature Backlog
